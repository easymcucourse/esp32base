## 🛠️ 实验目标：掌握 GPIO 的本质

在嵌入式开发中，最基础的操作就是控制一个引脚输出高低电平（点灯），或者读取一个引脚的状态（按键）。本实验通过一个简单的“联动”逻辑，展示了 GPIO 的完整闭环。

### 1. 🔍 核心概念：数字输入的三种模式
实验代码中展示了三种不同的输入配置模式，这是避坑的关键：
*   **`INPUT` (浮空模式)**: 引脚处于高阻态，极易受环境电磁干扰。在视频中你会看到，手靠近引脚甚至 LED 就会乱闪，这就是典型的“引脚浮空”现象。
*   **`INPUT_PULLUP` (内置上拉)**: 芯片内部连接一个电阻到 VCC。当没有外部信号时，引脚稳定在 **HIGH**。
*   **`INPUT_PULLDOWN` (内置下拉)**: 芯片内部连接一个电阻到 GND。当没有外部信号时，引脚稳定在 **LOW**。

### 2. ⚡ 硬件连接说明
*   **输入引脚**: GPIO 0 (对应代码中的 `INPUT_PIN`)。用于模拟传感器输入。
*   **输出引脚**: GPIO 21 (对应代码中的 `LED_PIN`)。连接 LED 灯，用于展示输出效果。

## 🚀 实验步骤

1.  **编译与上传**: 使用 Arduino IDE 或 PlatformIO 将 `L002_digital.ino` 上传至 ESP32。
2.  **观察浮空现象**: 保持 `pinMode(INPUT_PIN, INPUT)`，用手指靠近或触摸 GPIO 0，观察 LED 是否在无接触情况下异常闪烁。
3.  **对比上拉/下拉**: 分别取消第 17 行或第 18 行的注释，重新上传，观察引脚状态是否变得稳定。
4.  **串口监视器**: 打开串口监视器（波特率 `115200`），实时观察系统的初始化状态。

## 📊 GPIO 模式对比表

| 模式 | 内部电阻状态 | 悬空时电平 | 适用场景 |
| :--- | :--- | :--- | :--- |
| **INPUT** | 断开 | 不确定 (浮空) | 外部已有强驱动信号 |
| **INPUT_PULLUP** | 连接到 VCC | 高 (HIGH) | 低电平触发的按键/传感器 |
| **INPUT_PULLDOWN** | 连接到 GND | 低 (LOW) | 高电平触发的电路 |
